# セッション管理

## 要件

### Why

- 認証済みユーザーだけが保護画面を利用できる状態を維持する。
- セッション失効時の再認証を強制し、認証の有効性を保つ。

### What

- 認証成功時に認証済みセッションを確立できる。
- セッション有効期限（30 日）超過時は未認証として扱える。
- ログアウトで認証済み状態を終了できる。
- 未認証ユーザーは保護画面へアクセスできない。

### 受け入れ条件

- 認証済みユーザーのみ保護画面へアクセスできる。
- 期限切れ後は再認証が必要になる。
- ログアウト後は保護画面へアクセスできない。

## 仕様

### How

- 発行
  - 既存ユーザーのログイン成功時と新規登録完了時に本セッショントークンを発行する。
  - セッション有効期限は `発行時刻 + 30日`。
- 検証
  - Cookie から本セッショントークンを取得する。
  - JWT 署名・トークン種別を検証する。
  - 永続ストレージ上の有効トークンとの一致を確認する。
  - 期限切れレコードは無効として扱う。
- 多重ログイン制御
  - 同一ユーザー・同一デバイスの既存セッションは新規ログイン時に置換する。
- ログアウト
  - Cookie の本セッショントークンを無効化する。
  - 永続ストレージ上の対応トークンも無効化する。

### 失敗時挙動

- 無効トークン、期限切れトークン、未登録トークンは未認証として扱う。
- 未認証で保護画面に到達した場合はログイン画面へ遷移させる。
