# ログインコード検証

## 要件

### Why

- メール受信可能者のみを本人として扱い、なりすましを防ぐ。
- 検証成功後に、既存ユーザーと未登録ユーザーを適切な導線へ分岐させる。

### What

- ユーザーはログインコードを入力して検証できる。
- システムはコード一致と有効期限を検証できる。
- システムは検証試行に回数制限を適用できる。
- 検証成功時、既存ユーザーは本セッション、未登録ユーザーは一時セッションへ遷移できる。

### 受け入れ条件

- 既存ユーザーは検証成功後に認証済み状態へ遷移する。
- 未登録ユーザーは検証成功後に登録導線へ遷移する。
- 不正コード、期限切れコードは受理されない。
- 同一 IP の検証試行は 15 分間で 3 回までとなる。

## 仕様

### How

- 判定順序
  - 1) コード形式検証（6 桁数値）
  - 2) レート制限判定（送信元と処理種別の組み合わせ）
  - 3) コード照合（入力コードを SHA-256 でハッシュ化して照合）
  - 4) 有効期限確認（現在時刻より後であること）
  - 5) ユーザー存在判定（メールアドレス一致で検索）
- 成功時
  - 対象コードを削除する（ワンタイム化）。
  - 端末識別情報を生成する。
  - 既存ユーザー: 本セッションを発行し Cookie 設定。
  - 未登録ユーザー: 一時セッションを発行し Cookie 設定。
- Cookie
  - `httpOnly=true`, `secure=true`, `sameSite=lax`, `expires` を設定する。
- 監査
  - コード照合処理の成功時と不一致時に、実行時刻、送信元、対象メールアドレス、成功可否を記録する。

### 失敗時挙動

- 形式不正: 失敗を返す。
- 不一致/期限切れ: 失敗を返す。
- レート制限超過: 失敗を返す。
