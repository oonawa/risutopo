# ログインコード送信

## 要件

### Why

- パスワードを持たない利用開始導線を提供し、ログイン開始の心理的負荷を下げる。
- 認証機能への連続試行を抑止し、攻撃耐性を維持する。

### What

- ユーザーはメールアドレスを入力してログインコード送信を要求できる。
- システムは妥当なメールアドレス形式のみ受け付ける。
- システムは入力されたメールアドレス宛に、有効期限付きログインコードを送信できる。
- システムは送信試行に回数制限を適用できる。

### 受け入れ条件

- 正常入力時、対象メールアドレスにログインコードが送信される。
- コード発行から 10 分を超えた場合、当該コードは検証に利用できない。
- 同一 IP の送信試行は 15 分間で 3 回までとなる。

## 仕様

### How

- 入力検証
  - メール形式を検証し、不正時は処理を終了する。
- レート制限
  - 送信元と処理種別の組み合わせを判定キーにする。
  - 直近 15 分の試行履歴を集計し、3 回以上なら拒否する。
- コード生成
  - 6 桁の数値コードを生成する。
  - 平文コードは永続化せず、SHA-256 ハッシュを保存する。
- 保存
  - 同一メールアドレスの既存コードを削除してから新規発行する。
  - 有効期限は `発行時刻 + 10分`。
- 送信
  - メール本文に平文コードを含めて送信する。
- 監査
  - 成功/失敗と実行時刻、送信元、対象メールアドレスを記録する。

### 失敗時挙動

- メール形式不正: 送信しない。
- レート制限超過: 送信しない。
- 外部送信失敗: 発行済みコードは削除して失敗を返す。
