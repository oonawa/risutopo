# 一時セッション

## 要件

### Why

- 未登録ユーザーがメール検証後に登録を完了できる、安全な暫定認証状態を提供する。

### What

- 未登録ユーザーのコード検証成功時に一時セッションを発行できる。
- 一時セッションは有効期限 15 分で管理できる。
- 一時セッション中のみ登録処理を許可できる。
- 登録完了後に一時セッションを失効できる。

### 受け入れ条件

- 有効な一時セッションを持つ未登録ユーザーのみ登録完了できる。
- 登録完了後は同一一時セッションを再利用できない。

## 仕様

### How

- 発行
  - 未登録ユーザーのログインコード検証成功時に一時セッショントークンを生成・保存する。
  - 有効期限は `発行時刻 + 15分`。
- 検証
  - 登録処理時に一時セッショントークンの存在・期限を確認する。
  - トークンに紐づくメールアドレスと登録入力メールアドレスの一致を確認する。
- 無効化
  - 登録完了時に一時セッショントークンを削除する。
  - 同時に本セッションを発行し、状態を認証済みに切り替える。

### 失敗時挙動

- トークン無効/期限切れ: 登録を拒否する。
- メール不一致: 登録を拒否する。
