# ユーザーリストID解決

## 要件

### Why

- ログインユーザーに紐づくリストを特定し、登録・取得系処理の起点を確立する。

### What

- システムはユーザー識別子から対応するリスト識別子を取得できる。
- システムはリスト未作成ユーザーを「未登録」として扱える。

### 受け入れ条件

- リストが存在するユーザーではリスト識別子が返る。
- リスト未作成ユーザーでは未登録を示す値が返る。

## 仕様

### How

- 参照
  - リスト管理テーブルをユーザー識別子で検索し、最初の一致を返す。
- 応答
  - 一致がある場合はリスト識別子を返す。
  - 一致がない場合は未登録を示す値を返す。

### 失敗時挙動

- 本処理は内部で例外ハンドリングを持たないため、DB障害時は呼び出し元へ例外が伝播する。
