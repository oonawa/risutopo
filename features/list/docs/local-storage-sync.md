# ローカルストレージ同期

## 要件

### Why

- クライアント側で映画リストを保持し、重複判定や即時表示を高速化する。
- 初回表示時にDBデータを取り込み、クライアントキャッシュを最新化する。

### What

- システムは作品情報をブラウザ内の保存領域へ追加できる。
- システムはDB取得結果でローカルストレージを置換できる。
- システムは現在のローカルキャッシュを読み出せる。

### 受け入れ条件

- 作品追加時、クライアント側キャッシュに作品が追加される。
- 同期処理成功時、クライアント側キャッシュはDB取得結果で置換される。

## 仕様

### How

- 追加
  - クライアントリポジトリ経由で作品一覧キャッシュへ追加反映する。
- 置換
  - 一覧取得用のサーバー処理を呼び出し、成功時にキャッシュ全体を取得結果へ置換する。
- 読み出し
  - 状態管理ストアから現在の作品一覧キャッシュを読み出す。
- 管理
  - 同期エラー文言はフック内部状態で保持する。
