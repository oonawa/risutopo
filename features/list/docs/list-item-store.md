# リストアイテム保存

## 要件

### Why

- ユーザーが視聴予定/視聴済みの作品をリストへ登録・更新できるようにする。
- 同一リスト内の作品情報を一貫した形式で保持し、後続の表示・検索処理に利用可能にする。

### What

- システムは作品をユーザーのリストへ新規追加できる。
- システムは登録済み作品の内容を更新できる。
- システムは保存後の作品情報を、画面側でそのまま再利用できる形で返せる。

### 受け入れ条件

- 有効なリストを指定した保存要求は成功する。
- 新規追加時はリスト上で一意に識別できる作品として登録される。
- 更新時は、同じリスト内の既存作品のみが更新対象になる。
- リストが存在しない場合や配信サービスを特定できない場合は失敗を返す。
- 作品に外部映画データが付与されている場合は、その詳細情報も保存対象に含まれる。

## 仕様

### How

- 事前検証
  - 指定されたリストが実在するかを確認する。
  - 入力された作品情報から配信サービスを特定できるかを確認する。
- 新規登録
  - 受け取った作品識別子が同一リスト内に存在しない場合は、新規作品としてリストへ追加する。
  - 作品識別子が未指定の場合は、保存時に新しい識別子を生成する。
  - 保存する情報には、作品名、視聴先URL、配信サービス、視聴状態、登録時刻を含める。
- 更新
  - 同一リスト内に同じ作品識別子を持つ作品が存在する場合は更新する。
- 作品詳細情報の扱い
  - 公式タイトル、ポスター画像、背景画像、上映時間、公開年、監督名、外部データベース上の作品ID、あらすじなどの詳細情報が入力に含まれる場合は、それらも保存対象に含める。
  - 詳細情報が入力に含まれない場合は、基本情報のみで保存する。
- 応答
  - 保存後は、リスト表示に必要な作品情報一式を返す。
  - 詳細情報が保存対象に含まれていた場合は、応答にも同じ詳細情報を含める。

### 失敗時挙動

- リスト未存在: 失敗を返す（`映画リストが見つかりません。`）。
- 配信サービス未解決: 失敗を返す（`対象の配信サービスが見つかりません。`）。
- DB制約違反・永続化例外: 失敗を返す（`映画の追加に失敗しました。`）。
